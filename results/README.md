# Results — provenance

These files are a **fixed snapshot** of the WESAD benchmark (subjects S2–S17), generated by
`make reproduce` on the full dataset. WESAD is not redistributed (see
[../data/raw/README.md](../data/raw/README.md)), so a clean clone cannot regenerate them without first
downloading WESAD; `make demo` reproduces the same pipeline on synthetic data instead.

| File | Produced by |
| ---- | ----------- |
| `metrics.json`, `{binary,multiclass}_model_comparison.csv`, `{binary,multiclass}_per_subject.csv`, `shap_top_features.csv` | `scripts/run_experiment.py` |
| `ablation.csv` | `scripts/ablation.py` |
| `wrist.json` | `scripts/wrist.py` |
| `cross_dataset.json` | `scripts/cross_dataset.py` |
| `stats.json` | `scripts/stats.py` |

**Not committed** (regenerate on WESAD): `calibration.json`, `personalization.json`, `tuning.json`
and their figures — these require the WESAD download. Until then the paper's §4.7–4.8 tables are
placeholders.

> **Known staleness:** `metrics.json` was generated before the optimism-gap calculation was refined
> to use matched, non-overlapping windows (now in `scripts/run_experiment.py`, which writes a
> `loso_matched_accuracy` field). Re-running on WESAD adds that field and may shift the reported gap
> slightly — the live code is the source of truth.
